Git with Collaboration

The biggest benfit of GitHub is its ability to allow large amounts of people to alter code all at once. This is done by when a finished project is "pushed" that means that the older version is kept but the newer one is looked at as the new file with the person who altered it has their information stored. This information includes name, time of alteration, and other data that might be viewed as relevant. When documents are submitted at the same time if alters are done to a specific part in both files the user will have to choose which option they keep, other than that the VCS (Version Control System) will merge all the edits to one document.
